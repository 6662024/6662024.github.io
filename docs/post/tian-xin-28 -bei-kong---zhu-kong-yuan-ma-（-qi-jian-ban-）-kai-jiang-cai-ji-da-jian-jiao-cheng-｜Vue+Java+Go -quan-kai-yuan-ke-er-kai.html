一、系统概述
天信28 旗舰系列提供完整的被控端与主控端（总控）源码，支持与配套开奖网的自动同步。系统内置坐席客服 即时通讯（低延迟），后台含 双套模板 可自由切换，终端一键搭建，开箱即用，支持二次开发与深度定制。

建议环境：CentOS 7.8（COS7.8），被控与主控 必须分两台服务器 部署。
同步能力：配套开奖网可预设开奖并自动同步。
通讯体验：内置坐席客服，消息实时达达，无明显延迟。
模板系统：后台自带两套 UI 模板，可一键切换风格。
二、技术栈与架构
前端（业务端）：Vue + Vant（移动端体验优化）
后台（业务端）：Java + Vue + Element UI
数据/服务：Golang + Python 参与采集、任务与协同
总控平台：Java + Vue + Element UI（统一管控）
总控核心能力：查看未经授权 IP、关闭未经授权服务器、进入未经授权后台、添加授权 IP、添加授权房间、管理房间与游戏配置等。

推荐部署拓扑
主控（总控）节点：统一授权与策略、采集任务调度、可视化管理。
被控节点：按房间/玩法承载业务与前端服务，按授权策略接入。
配套开奖网：预设开奖源，主控定时或实时同步。
三、功能亮点
被控/主控分离：物理隔离，降低耦合与运维风险。
开奖采集：支持预设与自动同步，近期开奖趋势可视化。
授权与风控：总控可对 IP/房间/游戏配置 进行精细化授权与封禁。
即时客服：内置坐席系统，消息实时送达，提升用户沟通效率。
模板切换：后台两套风格随时切换，支持 Logo/色板定制。
一键终端：脚本化部署，自动拉起前后端与采集服务。
全开源：无加密，便于二开、代码审计与 CI/CD 集成。
四、环境要求与准备
操作系统：CentOS 7.8（建议最小化安装）
运行时：JDK 8+/11+、Node.js 16+/18+、Go 1.20+、Python 3.8+
基础服务：Nginx、Redis、MySQL（或兼容型数据库）
端口策略：放通主控 ↔ 被控 ↔ 开奖网必要端口（建议走内网专线/VPC）
